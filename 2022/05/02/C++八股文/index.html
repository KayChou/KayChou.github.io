C++八股文

发表于 2022-05-02 | 分类于 C++

C++ 基础知识及面试中常见问题。

C 和 C++ 区别

C 和 C++ 在语法上没有太大的区别。但是 C++ 在 C 的基础上新增了一些语法和关键字。

  • 新增语法和关键字

    C++允许定义自己的名称空间。语法方面的不同表现在如内存开辟与释放：new – malloc, delete – free ，以及输入输出 printf – cout 等。

  • C++ 有重载和虚函数的概念

    C++ 支持函数重载，而 C 不支持，因为 C++ 标识一个函数的方式是 函数名 + 参数列表，而 C 只通过函数名来标识函数。

    此外，C++ 还有虚函数的概念，可以用来实现多态。

  • C++ 有类的概念，但是 C 中只有结构体

  • C++ 增加了模板代码，并提供了强大的 STL 标准库。

C++基础知识

智能指针

智能指针的作用的管理指针，目的是为了避免申请的空间在函数结束时忘记释放，造成内存泄露等情况。智能指针本质上是一个类，当超出了类的作用域，类会自动调用析构函数，从而释放资源。

C++有四种智能指针

智能指针 描述
auto_ptr C++98 的方案，C11 中已经抛弃。存在潜在的内存崩溃问题，例如声明一个新指针 ptr2 指向已有指针 ptr1，当访问 ptr1 时会报错。
是因为允许多个智能指针指向同一个对象，当有一个指针释放后，其他指针会引发崩溃。
unique_ptr 替换了 auto_ptr, 保证同一时间只有一个智能指针可以指向对象。
shared_ptr 实现了共享式拥有概念，多个指针可以指向同一个对象
资源会在最后一个引用被销毁时释放。内部使用计数器机制表明资源被几个指针共享。
weak_ptr 不控制对象声明周期，而是指向 shared_ptr 管理的对象，由 shared_ptr 管理对象的内存。
为了解决 shared_ptr 的死锁问题，即当两个 shared_ptr 互相引用时，两个指针的计数永远不可能降为0。

C++内存分配情况

内存分类 描述
栈 存储局部变量，函数参数等。由编译器进行管理和回收
堆 由程序猿管理，手动通过 new, malloc, delete ,free进行分配和回收
空间较大，可能出现内存泄露、空闲碎片等情况
全局/静态存储区 存储初始化和未初始化的全局变量和静态变量
常量存储区 存储常量，一般不允许修改
代码区 存储承运的二进制代码

堆和栈的区别

堆 栈
由编译器管理，自动分配和回收。
一般保存的是局部变量和函数参数
连续的内存空间，空间较小		 由程序猿手动管理，通过 new malloc delete free 等进行回收
不连续空间，空间较大，较为灵活

1 申请方式

  • 栈：由系统自动分配。例如在声明函数的一个局部变量int b，系统自动在栈中为b开辟空间。
  • 堆：需要程序员自己申请，并指明大小，在C中用malloc函数；在C++中用new运算符。

2 申请后系统的响应

  • 栈：只要栈的剩余空间大于所申请的空间，系统将为程序提供内存，否则将报异常提示栈溢出。
  • 堆：操作系统有一个记录空间内存地址的链表，当系统收到程序的申请时，会遍历链表，寻找第一个空间大于所申请空间的堆节点，然后将节点从内存空闲节点链表中删除，并将该节点的空间分配给程序。对于大多数操作系统，会在这块内存空间中的首地址处记录本次分配的大小，这样，代码中的delete语句才能正确的释放本内存空间。另外，由于找到的对节点的大小不一定正好等于申请的大小，系统会自动地将多余的那部分重新放入到链表中。

3 申请大小的限制

  • 栈：在Windows下，栈是向低地址拓展的数据结构，是一块连续的内存的区域。站定地址和栈的大小是系统预先规定好的，如果申请的内存空间超过栈的剩余空间，将提示栈溢出。
  • 堆：堆是向高地址拓展的内存结构，是不连续的内存区域。是系统用链表存储空闲内存地址的，不连续。

4 申请效率的比较

  • 栈：由系统自动分配，速度较快。但程序员无法控制。
  • 堆：由new分配的内存，一般速度比较慢，而且容易产生内存碎片，不过用起来方便。 拓展：在Windows操作系统中，最好的方式使用VirtualAlloc分配内存。不是在堆，不是在栈，而是在内存空间中保留一块内存，虽然用起来不方便，但是速度快，也很灵活。

5 堆和栈的存储内容

  • 栈：在函数调用时，第一个进栈的是主函数的中的下一条指令（函数调用的下一个可执行语句）的地址，然后是函数的各个参数。在C编译器中，参数是由右往左入栈的，然后是函数的局部变量。静态变量不入栈。
  • 堆：一般是在堆的头部用一个字节存放堆的大小。堆中的具体内容由程序员安排。 数据结构方面的堆和栈与上边叙述不同。这里的堆是指优先队列的一种数据结构，第一个元素有最高的优先权；栈实际就是满足先进后出的性质的数学或数据结构。

总结：

  • （1）heap是堆，stack是栈；
  • （2）stack的空间由操作系统自动分配/释放，heap上的空间手动分配/释放；
  • （3）stack空间有限，heap是很大的自由内存区；
  • （4）C中的malloc函数分配的内存空间即在堆上，C++中对应的是new操作符。 程序在编译对变量和函数分配内存都在栈上进行，且内存运行过程中函数调用时参数的传递在栈上进行。

函数传递参数的方式

  • 值传递
  • 指针传递
  • 引用传递

值传递 vs 指针传递 vs 引用传递

  • 值传递：

    形参是实参的拷贝，改变形参的值不会影响外部实参的值。参数的值只能传入，不能传出。

  • 指针传递：

    指针传递传入的形参是实参的地址，也就是形参和实参都指向同一个地址，对形参进行指向区域的内容修改后，实参也发生相应变化。

    指针本质上是值传递，只不过形参和实参都是地址值，形参是实参的副本，如果修改形参的地址值，那么会指向一个新的空间，但是对实参不会产生影响。

  • 引用传递：

    形参相当于实参的别名，对形参的操作也是对实参的操作。

    在引用传递过程中，形参虽然也作为局部变量在栈中开辟了内存，但实际上存放的是实参变量的地址。被调函数对形参的访问和操作都会处理成间接寻址，即通过访问栈中存放的地址，去操作实参变量。

指针和引用的区别

总结：指针是一个实体，内容是指向某块内存的地址，而引用是一块内存的别名。

程序编译时：将指针和引用都添加到符号表中。对于指针，是将”指针名 - 指针变量地址” 添加到符号表中，因此指针包含的内容可以个改变，允许拷贝和赋值；对于引用，是将 ”引用变量名 - 引用对象地址“ 添加到符号表，不能修改。所以引用从始至终只能绑定在一块内存上，不能修改。

new / delete, malloc / free 的区别

都是用来在堆上分配和回收空间。new / delete 是操作符，而 malloc / free 是库函数。

执行 new 包括两个过程:

  • 分配未初始化的内存空间（使用 malloc进行分配）
  • 调用构造函数对空间进行初始化，返回空间的起始地址（如果第一步出错，则在第二步构造函数时出现异常，自动调用 delete 释放）

执行 delete 包括两个过程：

  • 调用析构函数对对象进行析构
  • 回收内存空间（free）
new / delete malloc / free
空间 初始化后的空间 未初始化的空间
区别 在创建的时候自动执行构造函数，消亡时自动析构 无法满足动态对象的要求

define 和 const 的区别

  • define 是在预编译阶段进行处理，没有类型，也没有类型检查。运行时不为宏定义分配内存
  • const 在编译期间进行处理，const 有类型，也有类型检查，程序运行时为 const 分配内存。

C++面向对象的三大特性：封装，继承，多态

特性
封装 封装成抽象的类，并提供了不同级别的保护（public, private, protected）
继承 让一个对象获得另一个对象的属性和方法。基类/父类，子类/派生类
多态 一个接口实现多种方法。

多态的实现

多态一般是值继承 + 虚函数实现的多态。如果将重载也看作多态的一种，则可以说 C++ 的多态分为静态多态和动态多态。

静态多态就是指重载，静态多态在编译时就决定了调用哪个函数，根据参数列表来决定；

动态多态是指通过子类重写父类的虚函数来实现的，在运行的时候决定调用哪个函数。一般来说当只提到多态的时候所指的就是动态多态。

动态多态的实现与虚函数表、虚函数指针相关。子类继承父类时，如果父类中有纯虚函数，则必须进行重写。

虚函数表，虚函数指针，虚函数的实现原理

当一个类中包含虚函数时，编译器会为该类生成一个虚函数表，保存类中虚函数的地址。当派生类继承时，也会为派生类生成虚函数表。

虚表是属于类的，一个类只有一个虚表，同一个类的所有对象都使用一个虚表。

当实例化一个派生类对象时，编译器检测到有虚函数，进而会为该对象生成一个虚函数指针，指向该类型的虚函数表。虚函数指针的初始化在构造函数中完成。

当调用虚函数时，会根据所指的真正对象的虚函数表指针去寻找虚函数地址，从而实现多态。
